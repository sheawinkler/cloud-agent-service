Cloud Agent Service MVP
=======================

Purpose:
This directory implements an MVP for a cloud coding-agent application. It
accepts a user request, validates it, upgrades the prompt, creates a job, runs a
repo-editing agent workflow, tests changes, applies policy gates, records
evidence, and syncs review output through local mock, generic Git, or GitHub App
paths.
Terminal runs are indexed in `lab_runs` so model/agent/harness promotion
outcomes can be queried and summarized.
Task suites, runtime status, optional auth/quotas, a lab UI, and ECS dry-run
dispatch plans are present as productization scaffolding. The harness registry
pre-indexes current agent harnesses, exposes a top_20 slice, and accepts
`custom:<name>` harness IDs as dispatch contracts.
Harness execution goes through an adapter ABI, attaches a per-harness security
profile, writes replayable run artifacts, and feeds a model/agent/harness
leaderboard.

Important boundary:
The current implementation does not call AWS and does not perform a real
deployment. Local PR and deployment outputs are artifacts used to prove the
control flow. Generic Git jobs push review branches to origin. GitHub App jobs
require app credentials before claiming live clone/push/PR behavior.
OpenAI Responses-backed agent runs require explicit enablement and an API key;
the default path is still deterministic and offline.
Ranked and custom harness IDs do not mean those third-party CLIs or managed
agents execute; a worker image or adapter must implement the selected harness
before live execution can be claimed.
The default local-template adapter is deterministic. The pi-coding-agent
adapter is opt-in via AGENT_CLOUD_ENABLE_PI_CODING_AGENT or
AGENT_CLOUD_ENABLE_EXTERNAL_HARNESS plus AGENT_CLOUD_PI_CODING_AGENT_CMD.

Mirendil framing:
Treat each repo update as a minimal model lab run: ModelSpec + AgentSpec +
HarnessSpec + bounded task + evidence -> PromotionDecision. The current default
model is local-deterministic; do not claim external SLM/LLM inference unless
wired and verified.

Primary files:
- app.py: FastAPI API.
- pipeline.py: end-to-end application flow.
- store.py: SQLite job and event storage.
- orchestrator.py: local queue and runner.
- worker.py: one-job container entry point.
- cloud_dispatch.py: ECS/Fargate dry-run dispatch request builder.
- harness_registry.py: ranked agent harness index, top_20 slice, and custom
  harness contracts.
- harness_adapters.py: adapter ABI plus local-template and opt-in Pi adapters.
- security_profiles.py: harness command, secret, path, network, and runtime
  boundaries.
- artifact_schema.py: run-artifact.json, transcript, diff, and artifact gates.
- task_corpus.py: shared replayable task corpus.
- compose.yaml: Docker Compose for local API/worker builds.
- EVALUATION.md: product and operational evaluation criteria.
- examples/agent_contract.json: worker payload and final result example.
- demo.sh: one-command local demo.
- scripts/demo_local_flow.py: no-cloud happy path demo.
- scripts/evaluate_mvp.py: single golden task evaluator.
- scripts/evaluate_task_suite.py: multi-run lab evaluator.
- scripts/install_allowed_modules.sh: dependency allowlist installer.
- tests/test_cloud_agent_service_flow.py: deterministic flow tests.

Expected successful flow:
1. Receive prompt.
2. Validate prompt, repo path, budgets, and runtime limits.
3. Select and record the harness contract.
4. Upgrade prompt into a concise implementation brief.
5. Create job record and queue event.
6. Dispatch local worker/container contract.
7. Copy or clone repo into an isolated workspace.
8. Record lab run config: model_id, agent_id, and harness_id.
9. Execute deterministic or selected adapter coding action.
10. Run compile/test checks.
11. Apply policy gates.
12. Record preview/browser/run-artifact evidence.
13. Sync review output and return final status plus promotion decision.
14. Index the terminal run for lab summary and task-suite comparison.

Do not:
- Treat mock PR URLs as real GitHub PRs.
- Treat local-deterministic as live external model inference.
- Treat local deployment artifacts as production deploys.
- Treat ECS dispatch plans as submitted AWS tasks.
- Treat indexed harness IDs as live adapter execution without worker proof.
- Commit runtime SQLite databases, generated workspaces, or artifacts.
- Add package dependencies without updating the allowlist and tests.
- Log or persist secrets.
