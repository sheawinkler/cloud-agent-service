Cloud Agent Service MVP
=======================

Purpose:
This directory implements an MVP for a cloud coding-agent application. It
accepts a user request, validates it, upgrades the prompt, creates a job, runs a
repo-editing agent workflow, tests changes, applies policy gates, records
evidence, and syncs review output through local mock, generic Git, or GitHub App
paths.
Terminal runs are indexed in `lab_runs` so model/agent/harness promotion
outcomes can be queried and summarized.
Task suites, runtime status, optional auth/quotas, a lab UI, ECS dry-run
dispatch plans, and env-gated ECS submit are present as productization scaffolding. The harness registry
pre-indexes current agent harnesses, exposes a top_20 slice, and accepts
`custom:<name>` harness IDs as dispatch contracts.
Harness execution goes through an adapter ABI, attaches a per-harness security
profile, writes replayable run artifacts, and feeds a model/agent/harness
leaderboard.
Analysis cases, experiment reports, SLM dataset export, router decisions,
worker callbacks, artifact refs, experiment batches, and dataset lineage turn
the control loop into a small Language Model Lab surface.
DuckDB lab warehouse, optional Postgres adapter, signed callback contracts,
worker leases, forge-provider status, Vercel preview deployment,
execution-provider status, and provenance manifests are now provider-layer
contracts around that loop.
Readiness scorecards, a local doctor CLI, and signed/idempotent event intake
make production cutover gaps and webhook-triggered jobs explicit. A cutover
rehearsal proves signed callbacks, signed event intake, and redacted ECS dry-run
plans locally without live cloud calls.

Important boundary:
The current implementation calls AWS only through `/jobs/<job_id>/cloud-dispatch`
when `AGENT_CLOUD_ECS_SUBMIT_ENABLED=1` and ECS env is configured. It does not
perform a real deployment. Local PR and deployment outputs are artifacts used to prove the
control flow. Generic Git jobs push review branches to origin. GitHub App jobs
require app credentials before claiming live clone/push/PR behavior.
OpenAI Responses-backed agent runs require explicit enablement and an API key;
the default path is still deterministic and offline.
Ranked and custom harness IDs do not mean those third-party CLIs or managed
agents execute; a worker image or adapter must implement the selected harness
before live execution can be claimed.
The default local-template adapter is deterministic. The pi-coding-agent
adapter is opt-in via AGENT_CLOUD_ENABLE_PI_CODING_AGENT or
AGENT_CLOUD_ENABLE_EXTERNAL_HARNESS plus AGENT_CLOUD_PI_CODING_AGENT_CMD.
The OpenAI edit adapter is opt-in via AGENT_CLOUD_ENABLE_OPENAI_EDIT_ADAPTER or
AGENT_CLOUD_ENABLE_EXTERNAL_HARNESS plus OPENAI_API_KEY.
SQLite is still the default operational store. DuckDB is the lab warehouse when
available and can also be used for local operational-store testing with
AGENT_CLOUD_DB_PROVIDER=duckdb. Postgres/RDS requires
AGENT_CLOUD_DB_PROVIDER=postgres, AGENT_CLOUD_POSTGRES_DSN, and a verified
psycopg runtime.
Vercel preview deployment is a dry-run contract unless
AGENT_CLOUD_DEPLOYMENT_PROVIDER=vercel_preview,
AGENT_CLOUD_VERCEL_DEPLOY_ENABLED=1, and VERCEL_TOKEN are configured.
Vercel Sandbox is an execution-provider status contract until a live adapter is
implemented and verified.
Event intake is unsigned local mode unless AGENT_CLOUD_EVENT_INGEST_SECRET is
configured and incoming requests include a valid x-agent-cloud-event-signature.
Cutover rehearsal endpoints/scripts are local proof only; they do not submit
ECS tasks, push Git branches, deploy, or clear readiness blockers.

Mirendil framing:
Treat each repo update as a minimal model lab run: ModelSpec + AgentSpec +
HarnessSpec + bounded task + evidence -> PromotionDecision. The current default
model is local-deterministic; do not claim external SLM/LLM inference unless
wired and verified.

Primary files:
- app.py: FastAPI API.
- pipeline.py: end-to-end application flow.
- store.py: operational job, event, callback, lease, and lab-run storage.
- database.py: SQLite/DuckDB embedded adapter plus optional Postgres adapter.
- callback_auth.py: worker callback HMAC token contract.
- event_ingest.py: signed/idempotent webhook and event intake contract.
- readiness.py: feature readiness scorecard and production cutover blockers.
- cutover.py: deterministic cutover rehearsal proof.
- forge.py: generic Git and provider-native review capability contracts.
- lab_warehouse.py: DuckDB materialized lab read model.
- orchestrator.py: local queue and runner.
- worker.py: one-job container entry point with optional HTTP callbacks.
- cloud_dispatch.py: ECS/Fargate dry-run dispatch request builder and env-gated
  live submitter.
- deployment.py: local mock and Vercel preview deployment providers.
- execution.py: local/ECS/Vercel Sandbox execution-provider contracts.
- provenance.py: run-level provenance manifest writer.
- artifact_store.py: local/S3 artifact reference indexing.
- harness_registry.py: ranked agent harness index, top_20 slice, and custom
  harness contracts.
- harness_adapters.py: adapter ABI plus local-template and opt-in Pi/OpenAI
  adapters.
- security_profiles.py: harness command, secret, path, network, and runtime
  boundaries.
- artifact_schema.py: run-artifact.json, transcript, diff, and artifact gates.
- task_corpus.py: shared replayable task corpus.
- analysis_lab.py: seeded analysis cases and experiment reports.
- dataset_export.py: redacted SLM JSONL export from replay artifacts.
- lab_router.py: leaderboard-backed routing decisions.
- compose.yaml: Docker Compose for local API/worker builds.
- EVALUATION.md: product and operational evaluation criteria.
- examples/agent_contract.json: worker payload and final result example.
- demo.sh: one-command local demo.
- scripts/demo_local_flow.py: no-cloud happy path demo.
- scripts/evaluate_mvp.py: single golden task evaluator.
- scripts/evaluate_task_suite.py: multi-run lab evaluator.
- scripts/doctor.py: local readiness/cutover scorecard.
- scripts/rehearse_cutover.py: no-cloud production cutover rehearsal.
- scripts/install_allowed_modules.sh: dependency allowlist installer.
- docs/sota-feature-readiness.md: comprehensive feature-readiness map.
- tests/test_cloud_agent_service_flow.py: deterministic flow tests.

Expected successful flow:
1. Receive prompt.
2. Validate prompt, repo path, budgets, and runtime limits.
3. Select and record the harness contract.
4. Upgrade prompt into a concise implementation brief.
5. Create job record and queue event.
6. Dispatch local worker/container contract.
7. Copy or clone repo into an isolated workspace.
8. Record lab run config: model_id, agent_id, and harness_id.
9. Execute deterministic or selected adapter coding action.
10. Run compile/test checks.
11. Apply policy gates.
12. Record preview/browser/run-artifact evidence.
13. Sync review output and return final status plus promotion decision.
14. Index the terminal run for lab summary, task-suite comparison, analysis
    reports, dataset export, and router recommendations.
15. Index artifact references and accept worker callbacks when a cloud worker
    posts them.
16. Record deployment/execution provider evidence and a provenance manifest.
17. Report readiness and accept signed/idempotent event intake when configured.
18. Rehearse cutover locally without live cloud calls.

Do not:
- Treat mock PR URLs as real GitHub PRs.
- Treat local-deterministic as live external model inference.
- Treat local deployment artifacts as production deploys.
- Treat ECS dispatch plans as submitted AWS tasks.
- Treat ECS submit as live unless `AGENT_CLOUD_ECS_SUBMIT_ENABLED=1` and the
  submit endpoint returned a submitted dispatch.
- Treat indexed harness IDs as live adapter execution without worker proof.
- Treat DuckDB as production database infrastructure.
- Treat Postgres/RDS as live without AGENT_CLOUD_DB_PROVIDER=postgres,
  AGENT_CLOUD_POSTGRES_DSN, and a verified psycopg runtime.
- Treat callbacks as authenticated without AGENT_CLOUD_WORKER_CALLBACK_SECRET.
- Treat event intake as public/signed without AGENT_CLOUD_EVENT_INGEST_SECRET
  and a valid x-agent-cloud-event-signature.
- Treat Vercel preview/Sandbox contracts as live without env flags and evidence.
- Treat cutover rehearsal success as production readiness.
- Commit runtime SQLite databases, generated workspaces, or artifacts.
- Add package dependencies without updating the allowlist and tests.
- Log or persist secrets.
